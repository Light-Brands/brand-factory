# Light Brands: Master Strategic Plan

## The Two-Division Model for Scalable Brand Creation

---

## Executive Summary

### The Challenge

Light Brands has evolved from a vision into a functioning consultancy creating purpose-driven brands with ethical AI integration. However, the current structure conflates client-facing work with internal development, creating friction, scope creep, and unclear boundaries between sales/relationship management and pure engineering.

### The Solution: Clean Division Architecture

Light Brands operates as **two distinct but interconnected divisions**:

```
┌─────────────────────────────────────────────────────────────────────────────┐
│                           LIGHT BRANDS ECOSYSTEM                            │
├─────────────────────────────────┬───────────────────────────────────────────┤
│     LIGHT BRANDS CONSULTING     │       LIGHT BRANDS DEVELOPMENT            │
│     ─────────────────────────   │       ─────────────────────────           │
│                                 │                                           │
│  ► Client Acquisition           │  ► Engineering & Architecture             │
│  ► Sales & Proposals            │  ► Code Systems & Integrations            │
│  ► Relationship Management      │  ► AI Agent Development                   │
│  ► Project Scoping              │  ► Platform Building                      │
│  ► Client Dashboards            │  ► Internal Tools & Infrastructure        │
│  ► Payments & Invoicing         │  ► Quality Assurance                      │
│  ► Delivery Coordination        │  ► Technical Documentation                │
│                                 │                                           │
│  ALL CLIENTS EXIST HERE         │  NO CLIENTS ALLOWED HERE                  │
│                                 │                                           │
└─────────────────────────────────┴───────────────────────────────────────────┘
```

### Core Principle

**Paid work flows one direction:** Consulting closes deals → Development builds → Consulting delivers.

No client ever touches Development. No developer ever touches sales. Clean boundaries create clean outcomes.

│  │  • Brand guidelines document (PDF)                       │    │
│  └─────────────────────────────────────────────────────────┘    │
│                                                                  │
│  Investment: $8,500                                              │
│  Status: PAID → Ready for Development                            │
│                                                                  │
│  Acceptance Criteria:                                            │
│  □ Logo renders correctly at all sizes                          │
│  □ Colors pass WCAG AA contrast                                 │
│  □ Typography scales responsively                               │
│  □ Guidelines document complete and accurate                    │
│                                                                  │
│  Timeline: 10 business days from handoff                        │
│                                                                  │
└─────────────────────────────────────────────────────────────────┘
```

**Work Unit Lifecycle:**

1. **Scoping** (Consulting): Define deliverables, acceptance criteria, investment
2. **Agreement** (Consulting): Client approves scope and pays
3. **Handoff** (Consulting → Development): Work unit enters development queue
4. **Building** (Development): Engineering team executes
5. **QA** (Development): Quality assurance and testing
6. **Return** (Development → Consulting): Completed work returned
7. **Delivery** (Consulting): Client receives deliverable and provides feedback
8. **Closure** (Consulting): Acceptance confirmed, work unit closed

To be the engineering powerhouse where Light Brands are built—maintaining absolute focus on craft, quality, and innovation without client interruption.

```
